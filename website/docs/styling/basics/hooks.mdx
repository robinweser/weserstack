Hooks (not to be confused with [React hooks](https://react.dev/reference/react/hooks)) are what makes this package tick.<br />
Leveraging a [CSS Variable Trick](https://weser.io/blog/inline-styles-on-steroids#css-variable-trick), they enable us to use pseudo classes and at-rules like media queries with plain inline styles.

## How It Works

Nick, the creator of CSS Hooks, wrote a great [article](<https://weser.io/blog/%5Bhttps://nsaunders.dev/posts/css-madness-to-hooks%5D(https://nsaunders.dev/posts/css-madness-to-hooks/)>) where he explains the thought process behind creating css-hooks and how it works internally.
Here's a quick summary of the gist:

```css
* {
  --hover-off: initial;
  --hover-on: ;
}
:hover {
  --hover-off: ;
  --hover-on: initial;
}
```

```html
<div style="color: var(--hover-on, red) var(--hover-off, blue)">Hover me</div>
```

<Box marginTop={2}>
  <style
    dangerouslySetInnerHTML={{
      __html: `* {
  --hover-off: initial;
  --hover-on: ;
}
:hover {
  --hover-off: ;
  --hover-on: initial;
}`,
    }}
  />
  <Box
    padding={4}
    alignItems="center"
    justifyContent="center"
    bg="var(--colors-background-alternate)"
    style={{
      color: 'var(--hover-on, red) var(--hover-off, blue)',
      lineHeight: 1,
    }}
    dangerouslySetInnerHTML={{ __html: 'Hover me' }}
  />
</Box>

## Defining Hooks

Since this syntax is cumbersome to write, this package provides a way to define hooks in a more declarative way when using [createRenderer](/styling/api/createRenderer):

```ts
const hooks = {
  ':hover': 'hover',
  ':focus': 'focus',
  m: '@media (min-width: 800px)',
  l: '@media (min-width: 1024px)',
}
```
