This guide aims to give you a quick overview of the basics of this package and how to use it.<br />
Under the hood, this package uses [css-hooks](https://css-hooks.com) to create inline styles with pseudo classes and at-rules like media queries.

## Preparation

Before you get started, it is recommended to read at least the [basics](/styling/basics) section. This will give you a rough understanding of how the package works internally.

## Creating a Renderer

The first thing you need to do is create a renderer. This is the core of the package and will be used to create the CSS function and all the static CSS required to make it work.<br />
It accepts a `hooks` object amongst other optionsthat maps object keys to CSS pseudo classes or at-rules like media queries.

```ts
import { createRenderer } from '@weser/styling'

const [staticCSS, css] = createRenderer({
  hooks: {
    ':hover': 'hover',
    m: '@media (min-width: 800px)',
    l: '@media (min-width: 1024px)',
  },
})

export { staticCSS, css }
```

## Injecting the Static CSS

The static CSS needs to be injected into the document somewhere. This is usually done in your root or layout file.

```tsx
import { staticCSS } from './renderer'

function RootLayout({ children }: PropsWithChildren) {
  return (
    <html>
      <body>
        <style dangerouslySetInnerHTML={{ __html: staticCSS }} />
        {children}
      </body>
    </html>
  )
}
```

## Using the CSS Function

The CSS function is the main way to style your components. It accepts a style object and returns a resolved inline style object.

```tsx
import { css } from './renderer'

const style = css({
  color: 'red',
  fontSize: 20,
  m: {
    fontSize: 16,
  }
  ':hover': {
    color: 'blue',
  },
})

function Text() {
  return <div style={style}>Hello</div>
}
```

And that's it! You're now able to create styled components wherever you want.
