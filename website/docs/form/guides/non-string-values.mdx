---
title: Non-String Values
order: 7
---

By default, [useField](/form/api/useField) (and [useFormField](/form/api/useForm#useformfield) respectively) expects string values and defaults to an empty string if no initial value is provided.<br />
In order to also support e.g. `boolean` values or arrays, we can provide custom types.

<Callout title="Note">
  Passing a custom value type and change event will also change the type of
  `field.value` and the expected input for [update](/form/api/useField#update).
</Callout>

<Callout title="Tip">
  There's also the [formatValue](/form/api/useField#options) option to convert
  the value before it's passed to [props](/form/api/useField#props) and
  [inputProps](/form/api/useField#inputprops).
</Callout>

```tsx
const acceptsTerms = useField<boolean, React.ChangeEvent<HTMLInputElement>>(
  z.boolean(),
  {
    // alter how the value is obtained from the change event if neccessary
    // e.g. for checkboxes or custom inputs
    parseEvent: (e) => e.target.checked,
    // set an initial value overwritting the default empty string
    value: false,
  }
)

// custom multi-select input that returns an array of values on change
type Tags = Array<string>
type TagsChangeEvent = (value: Tags) => void

const tags = useField<Tags, TagsChangeEvent>(z.array(z.string()), {
  parseEvent: (value) => value,
  value: [],
})

// also reflects the update and value type
tags.update({
  value: [...tags.value, 'foo'],
})
```
